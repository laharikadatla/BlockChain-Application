# BlockChain-Application

This is a client-server project where clients will send messages to server and server will persist messages. For each 100 messages server will store messages in a block and implement a BlockChain. Each client and server will have identity, consists of {public, private} key-pair.

## Instructions to run

#### Clone the project,

```sh
  $ git clone git@github.com:laharikadatla/BlockChain-Application.git
```
#### Installation

1. Make sure [Python 3.6+](https://www.python.org/downloads/) is installed. 

2. Make sure you have Mongo installed and running in machine.

3. **Install requirements**
  ```sh
    $ pip install requirements.txt
  ``` 

4. **Run the client:**
  ```sh
    $ python client.py
  ```

5. **Run the server:**
  ```sh
    $ python server.py
  ```
  The application should be up and running at [http://localhost:5000](http://localhost:5000).
    
## About the application

1. **Client :**
      In the `client.py` we are going to create 10 clients which run simultaneously and will send signed messages to server
      * For Each client we are going to generate private and public key and store them in `keys` folder.
        ```sh
          random_generator = Random.new().read
          key = RSA.generate(2048, random_generator)
          privateKey, publicKey = key, key.publickey()
        ```
      
      * Each client is configured to generate and send a message every second, for 1 minute.
      * Message Schema:
        ```sh
          {'from':<client_id>,'amount':<amount>,'time':<current_time>}
        ```
      * Client will sign the message using privateKey and also encrypt the message using server's publicKey
        ```sh
          # sign message using client private key and using SHA-512
          signer = PKCS1_v1_5.new(client_private_key)
          digest = SHA512.new()
          digest.update(json.dumps(message).encode('utf-8'))
          signature = signer.sign(digest)
          
          # encrypt message using server's public key
          cipher = PKCS1_OAEP.new(serverPublicKey)
          encrypt_message = cipher.encrypt(json.dumps(message).encode('utf-8'))
          
        ```
      * Client will send encrypted_message, signature, client_id to server.
        ```sh
          {'client_id':<client_id>,'message':encrypted_message,'signature':signature}
        ```
        
 
2. **Server:**
      Server`server.py` will validate whether the request from client valid or not. If valid stores the message in Mongo and if messages count is equal to 100 then server will add those messages to block chain and mine the block.
      * Server will decrypt the message and will validate message schema.
          ```sh
            # decrypt message
            cipher = PKCS1_OAEP.new( server_private_key )
            decrypted_message = cipher.decrypt( b64decode( request_data['message'] ) )

            # verify signature is valid or not
            signer = PKCS1_v1_5.new(client_public_key)
            digest = SHA512.new() #By default considering 512 hash for now
            digest.update(decrypted_message)
            signer.verify(digest, signature)
          ```
      * Each validated message will be stored in Mongo and if messages count equal to 100 then will add those transaction to block and will mine_block
 
3. **Blockchain and Merkle Tree:**
      * BlockChain `blockchain.py` will be initialized while starting server and genesis block will be created.
      * BlockChain will have:
                `chain: The list of blocks, 
                open_transactions: The list of open transactions`
        
      * Block class represents a single block of our blockchain.
      * Block will have:
          ```sh
             hash: The signed merkle root hash of this block
             previous_hash: The hash of the previous block in the blockchain
             timestamp: The timestamp of the block (automatically generated by default)
             transactions: A list of transaction which are included in the block
             proof: The proof of work number that yielded this block

          ```
        For Each block will calcualte merkle root hash `merkletree.py` and will store the signed merkle root hash 
          ```sh
            signer = PKCS1_v1_5.new(server_private_key)
            mtree = MerkleTree(self.transactions)
            signed_hash = signer.sign(mtree.root.hash_obj)
          ```

4. **test:**
       `test.py` consists of test cases 
      **Run Tests:**
       ```sh
       $ python test.py
       ```
